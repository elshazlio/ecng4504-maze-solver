# 01100 Left-T Design

## Problem

`working_code.ino` currently requires `s0`, recent `s0`, or post-left assist before `01100` can count as a narrow left stem. That means a real left T can be invisible when the robot approaches slightly right-of-center and only produces `00100 -> 01100 -> 00100`.

## Chosen Design

Keep the existing strict handling for stale/free-floating `01100`, but add one extra context gate: if the previous sampled pattern was a clean centered `00100`, then a current `01100` should count as a left junction and return `DEC_SL`.

## Why This Design

- It fixes the exact confirmed blind spot without making every isolated `01100` a left turn.
- It is a self-contained single-file change in `working_code.ino`.
- It preserves the existing post-left assist and `s0Recently` logic rather than replacing it.

## Expected Behavior

- `00100 -> 01100` should commit `DEC_SL` even if `s0` never fired.
- Stale `01100` reached from some other prior pattern should still return `DEC_S` unless existing `s0` or assist gates apply.
- Existing left-junction and close-successive-left behavior should remain intact.

## Testing

- Add a regression test that proves `00100 -> 01100` now commits left.
- Keep the stale-`01100` test to prove the broader strictness remains.
- Run the targeted host-side regression binaries after the code change.
